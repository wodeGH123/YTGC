# YTGC

岩土工程资料文件仓库。

本仓库用于集中保存、整理和追踪岩土工程相关资料，包括项目资料、勘察资料、试验数据、设计计算、图纸、照片、规范标准和常用模板。

## 目录结构

```text
YTGC/
├─ 00_index/              # 总索引、资料台账、目录清单
├─ 01_projects/           # 按项目归档的资料
├─ 02_standards/          # 规范、标准、规程
├─ 03_investigation/      # 勘察资料、钻孔、地层、原位测试
├─ 04_laboratory_tests/   # 室内试验、土工试验、水质/土腐蚀性等
├─ 05_design_calculation/ # 设计计算书、分析表、计算模型
├─ 06_drawings/           # CAD、PDF 图纸和示意图
├─ 07_reports/            # 勘察报告、设计报告、审查意见
├─ 08_photos/             # 现场照片、过程记录图片
├─ 09_meetings/           # 会议纪要、沟通记录、往来文件
├─ 10_templates/          # 常用表格、报告模板、计算模板
└─ archive/               # 已归档、历史版本或不常用资料
```

## 建议命名规则

文件名建议使用：

```text
日期_项目简称_资料类型_版本.扩展名
```

示例：

```text
2026-07-15_某项目_勘察报告_v1.pdf
2026-07-15_某项目_钻孔柱状图_v1.dwg
2026-07-15_某项目_土工试验汇总_v1.xlsx
```

## 使用建议

- 重要资料优先保存 PDF 或原始可编辑文件。
- 大文件、扫描件、图纸等建议按项目单独建文件夹。
- 修改报告或计算书时使用版本号，例如 `v1`、`v2`、`revA`。
- 不建议上传临时文件、缓存文件、软件自动生成的中间文件。

## 快速开始

1. 在 `01_projects/` 下为每个项目建立独立目录。
2. 将项目资料按资料类型放入对应子目录。
3. 在 `00_index/materials-register.md` 中登记关键资料。
4. 重要变更及时提交并写清楚提交说明。
